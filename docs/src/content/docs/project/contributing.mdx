## Overview

Thanks for helping improve Committy! This guide summarizes how to contribute effectively.

## How to Contribute

1. Fork the repo on GitHub.
2. Create a feature branch from `develop`.
3. Make your changes.
4. Commit with a clear, conventional message (e.g., `feat(cli): add --foo flag`).
5. Push your branch to your fork.
6. Open a Pull Request targeting `develop`.

## Code Style

- Follow the existing style and conventions.
- Keep changes focused and cohesive.
- Ensure code is tested and does not break existing functionality.

## Reporting Issues

- Create an issue with a clear title and detailed description.
- Include steps to reproduce, expected vs actual behavior, and any logs if relevant.

## Pull Request Guidelines

- Keep PRs focused on a single change or feature.
- Provide a descriptive PR body and reference related issues (e.g., `#123`).
- Keep your branch up to date with `main` as needed.
